In order for Lucidchart to access your AWS infrastructure, you’ll need to give Lucidchart the credentials for a new IAM user with an inline policy.

Creating an AWS User for Lucidchart's AWS Import

See below to learn how to create an IAM user with an inline policy in your AWS console.

Create an IAM User

To create an IAM user with an inline policy for Lucidchart's AWS Import, follow these steps:

  1. Go to the AWS console and navigate to the “Users” section of Identity Access and Management
  1. Click “Create New Users” and enter a name for the user, like “Lucidchart_AWS_Import”. Be sure to check the box next to “Generate an access key for each user” — Lucidchart will use that access key to retrieve the information about your infrastructure.
  1. On the next screen, either click “Show User Security Credentials” and copy the Access Key ID and Secret Access Key for later use, or click “Download Credentials” to receive the Key and Secret as a .csv file. We recommend downloading the file so you can more easily reuse it during the import step.
  1. Click “Close”, and then click on your new user in the “Users” screen.
  1. Select the “Permissions” tab of the user summary, and expand the “Inline Policies” section to display a link to create a new inline policy.
  1. After creating a new inline policy, select “Custom Policy”, which lets you paste in the correct policy for Lucidchart’s AWS import. Click “Select” to open up the policy editor.
  1. Take the policy you retrieved from Lucidchart’s AWS Import tool (it’s on the first screen of the import workflow, which you can access from the “Import” button in the document list or the File > Import menu inside of a document), and paste it into the policy editor. Give the policy a meaningful name, like “Lucidchart_AWS_Import”.
  1. Click “Apply Policy” to apply the policy to the user, and you should see that the user summary now includes a new inline policy.

Now you’re ready to use this user as part of your Lucidchart import! If you haven’t already saved this user’s access key and secret, go to “Security Credentials”, click “Create Access Key”, and then download a new set of credentials for the user.
