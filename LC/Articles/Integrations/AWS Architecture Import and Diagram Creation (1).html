Import Via Docs List

To import your AWS architecture into Lucidchart, click on the “Import” button and select the “AWS Architecture” option.

Import Via API

To import your AWS infrastructure into Lucidchart via API, follow these steps:

  1. Create an IAM user with the stated policy. We suggest creating a new user for your Lucidchart import credentials and adding an inline policy to that user. This will allow you to easily control or revoke access, as needed.

  2. Enter the IAM keys for the newly created user manually, or upload a CSV or config file to enter the keys. Additionally, select the region you wish to use for the AWS import. Right now, Lucidchart only supports importing one region per document.

    • Note about security: Lucidchart does not save these credentials, and only uses them once at the time of import. You’ll have to re-add the credentials for every document you create. We hope this reduces security concerns, as your credentials are only being used at the time that you click “import”.
  3. Import the AWS architecture
AWS Diagram Creation

Once you have imported your AWS architecture, you can create your diagram using the imported objects. To create your diagram, follow these steps:

  1. Upon importing your AWS architecture, Lucidchart will generate a list of AWS components within the “Imported AWS Toolbox” section of the shape library menu. You can use the “Search” feature to search the component list by component name and/or tag. Additionally, you can filter the list to display specific types of components. 
  2. Drag and drop components from the list to create your diagram.
  3. After dragging out diagrams, click the “+” sign to examine the the shape’s connected components. You can filter the list in terms of how traffic flows from that shape: “incoming” lists all components that send traffic inbound to the selected component, and “outgoing” lists all components that receive traffic from the selected component.

  4. Click on a connected component to automatically draw and connect the components.

  5. When dragging out new components, Lucidchart will automatically draw connections between connected components based on your diagram settings (see “Import Settings” above).

FAQ

Why is there a connection between two items I didn't expect to be connected?

Lucidchart's AWS import draws connections based on security groups and subnets, and looks for ports to be open on both sides of a connection. If you're seeing connections you don't expect between items, it's likely that the items have open ports between them. For example, one item may have a port open to send traffic to all IPs, and the other item may have the same port open to receive traffic from all IPs.

Why don't you make the diagram completely automatic?

The first version of Lucidchart's AWS Import tool built diagrams automatically, but customer feedback showed that creating such large, complex diagrams wasn't helpful. The current iteration of the tool provides all the resources to choose from, including the correct icons, names, tags, and all other metadata. This allows a user to quickly create a diagram for their specific use case without any of the tedious work of typing out names or copying and pasting information from the AWS console.

Why are some of my instances not showing up?

Instances within autoscaling groups are represented by the autoscaling group itself, rather than showing each individual instance, as they would all have the same connections. You can still see the instance IDs for those instances in the "Shape Data" panel.

How do I give Lucidchart access to my AWS architecture?

In order for Lucidchart to access your AWS infrastructure, you’ll need to give Lucidchart the credentials for a new IAM user with an inline policy. For an overview of how to create an IAM user with an inline policy, check out this article.
